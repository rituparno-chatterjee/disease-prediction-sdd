# Training Pipeline Summary

## Scope
This document summarizes the implemented logistic regression training pipeline for:
- diabetes
- heart
- kidney

Primary modules:
- `src/training/train_logistic.py`
- `src/training/logistic_baseline.py`
- `src/training/summarize_metrics.py`

## Inputs
Training reads encoded datasets from `data/processed` using explicit target mapping:
- diabetes -> `Outcome`
- heart -> `num`
- kidney -> `class`

## Pipeline Flow
1. Load dataset and validate target availability.
2. Drop rows with missing targets.
3. Remove non-trainable columns:
- target column
- ID-like columns (`id`, `*_id`)
- kidney forced-drop leakage proxy (`affected`)
4. Run target leakage guard:
- scans integer-like feature columns
- flags near-perfect direct or inverse target matches
- hard-fails if proxy threshold is exceeded
5. Build stratified splits:
- train/validation/test = 70/15/15
- fixed random seed for reproducibility
6. Train model with GridSearchCV over `C`:
- diabetes and kidney: median impute -> standard scale -> logistic regression
- heart: numeric branch (median impute + scale) and categorical branch (mode impute + one-hot) via ColumnTransformer
- classifier uses `class_weight=balanced`
7. Evaluate on validation and test:
- binary: F1, precision, recall, ROC-AUC, PR-AUC, confusion matrix
- multiclass: macro F1/precision/recall, OvR macro ROC-AUC, confusion matrix
8. Persist artifacts and metadata.

## Artifacts
For each dataset under `models/<dataset>/`:
- `logistic_pipeline.joblib`
- `metrics.json`
- `feature_schema.json`
- `train_config.json`

Cross-dataset reports:
- `reports/training-metrics-summary.json`
- `reports/training-metrics-summary.md`

## Inference Pipeline
Inference uses the full saved training artifact per dataset:
- `models/<dataset>/logistic_pipeline.joblib`

This means preprocessing and model inference are both loaded from the trained pipeline artifact. For heart, this includes the type-aware `ColumnTransformer` path:
- Numeric: `SimpleImputer(strategy="median")` + `StandardScaler()`
- Categorical: `SimpleImputer(strategy="most_frequent")` + `OneHotEncoder(handle_unknown="ignore")`

Inference module:
- `src/inference/predict.py`

The inference implementation supports:
- single-record JSON inference
- batch inference from JSON list
- batch inference from CSV
- schema validation using `feature_schema.json`

## Inference Output Fields
Prediction output includes:
- selected dataset
- predicted class
- predicted class label (human-readable label for all supported datasets)
- class probabilities (when `predict_proba` is available)
- top class probability (max class probability)
- validation warnings
- model metric summary from `metrics.json` (when present)

## Heart Model Limitation
Each heart prediction includes this warning:

"This heart disease prediction is generated by a baseline logistic regression model. The model has modest macro F1 performance, so the result should be interpreted cautiously and not used for clinical decision-making."
