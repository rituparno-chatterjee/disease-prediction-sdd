High-Level Design

1. Raw DataCSV inputs for three datasets are loaded from data/raw.
2. PreprocessingToken cleanup, imputation, encoding, target cleanup, scaling.
3. Feature SetupExplicit target mapping and trainable feature selection.
4. TrainingDataset-specific pipeline + GridSearchCV over C.
5. ArtifactsSave model pipeline, schema, metrics, and train config.
6. InferenceValidated JSON/CSV input through saved pipeline artifacts.
7. UI + ReportsPreview metrics and prediction outputs from saved JSON files.
Interview framing: this system is a reproducible baseline ML prototype, not a clinical-grade decision support tool.

    Final 10-Minute Interview Plan

    1. State scope and datasets, then describe the end-to-end architecture from raw data to UI.
    2. Explain preprocessing order and why token standardization before encoding mattered.
    3. Show training design choices: leakage guard, dataset-specific pipelines, and metric strategy.
    4. Discuss artifacts and reproducibility: what is saved and why inference uses those exact artifacts.
    5. Walk through inference validation and response structure, including label mapping and warnings.
    6. Acknowledge current limitations: heart multiclass performance and non-clinical scope.
    7. Close with concrete next-step improvements: data volume, calibration, external validation, stronger models.
